# Phase 4: AI Workflow Layer

## Objective
Turn AI enrichment into a controllable workflow instead of a black box.

## Scope
- Confidence scoring
- Review endpoints
- Editable AI outputs
- Review queues
- Structured action item management

## Workstreams
1. Confidence model for categories/entities/action items
2. AI output approval and override APIs
3. Review queue generation
4. Action item persistence and status management
5. Auditability for AI vs user changes

## Deliverables
- Reviewable AI outputs
- “Needs review” queue
- Action items as first-class objects
- User trust improvements

## Exit criteria
- Users can approve, reject, or edit AI results
- Review queue is populated by clear rules
