# Phase 2: Search & Retrieval Core

## Objective
Make search the product’s strongest and most trustworthy feature.

## Scope
- Hybrid search correctness
- Ranking improvements
- Saved searches
- Suggestions and popular searches refinement
- Related documents experience
- Search performance tuning

## Workstreams
1. Shared filter application helper
2. Hybrid merge/ranking cleanup
3. Search API contract refinement
4. Suggestions and popular search tuning
5. Similar/related/duplicate result separation
6. Search pagination and latency improvements

## Deliverables
- Consistent search results across modes
- Related documents surfaced clearly
- Better search UX hooks for frontend
- Performance baseline established

## Exit criteria
- No search filter leakage
- Related results are meaningful
- Search response time targets met on representative dataset
